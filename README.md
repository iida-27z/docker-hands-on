# Docker勉強会

## はじめに

### 1. このドキュメントについて

このドキュメントは、これからDockerを触り始める方向けのハンズオン教材です。単にコマンドを実行するだけでなく、Dockerがどのような課題を解決する仕組みで、なぜ開発や運用の現場で広く使われているのかを理解しながら進めることを目的としています。

### 2. そもそもDockerとは何か

Dockerは、アプリケーションとその実行に必要な設定や依存関係をひとまとめにし、どこでも同じように動かせる形にしたコンテナ型の仮想環境を扱うためのツールです。コンテナは、他の方式の仮想マシンと比べて、より軽量で高速に起動できる特徴があります。

### 3. なぜDockerを使うのか

Dockerが使われる大きな理由は、「自分の環境では動くのに、他の人の環境では動かない」という問題を減らせる点です。コンテナは、アプリケーションの動作に必要なOSレベルの設定からライブラリ、ツール類までをまとめて管理できるため、環境の違いによるトラブルを抑えやすくなります。また、本番環境をDockerで構築することで、開発環境と本番環境の差異を最小化することも可能です。さらに、Dockerの強みの一つとして、`Dockerfile`や`docker-compose.yml`といった設定ファイルで環境構築の手順をコード化できる点があります。

### 4. Dockerと深く関わる重要概念

Dockerを理解する際は、コマンド操作だけでなく、Dockerの背後にある考え方を押さえることも重要です。特に次の2つは、押さえておくべき主要な概念です。

- **実行環境のコンテナ化**:
アプリケーションをホスト環境からある程度切り離し、必要な実行条件をまとめて持ち運べるようにする考え方です。これにより、軽量でありながらも、再現性と可搬性が高い仮想環境を提供し、環境差分による問題を減らすことができます。

- **インフラのコード化**:
サーバーや実行環境の構成を手順書ではなくコードや設定ファイルとして管理する考え方で IaC (Infrastructure as Code) とも呼ばれます。Dockerでは、Dockerfileやdocker-compose.ymlのような仕組みを通じて、環境構築の再現性を高め、変更履歴の追跡やレビューもしやすくなります。

### 1. コンテナを起動してみる

#### Dockerのインストール確認
まずは、この環境で問題なくDockerが動作するかを確認しましょう。以下のコマンドをターミナルから実行して、Dockerのバージョンが表示されればOKです。
```bash
docker --version
```
また、この後のコマンドでDockerがどのように動作しているのかを理解するために、VS Codeの画面左側のアクティビティバーから`Containers`を選択しておいてください。

#### イメージの取得
次に、コンテナ環境のテンプレートとなるイメージをインターネット上のDocker Hubから取得します。今回は、`Ubuntu`というLinux環境の最新版イメージを以下のコマンドから取得します。イメージは保管されている先とそのバージョンを`リポジトリ名:タグ名`で指定しますが、タグ名を`latest`と指定すると、最新版のイメージが取得されます。
```bash
docker pull ubuntu:latest
```
実行完了後には、VS Codeの`Containers`の画面で、`ubuntu`というイメージが追加されていることを確認してください。このイメージを基にして、コンテナ仮想環境が起動できます。

#### コンテナの起動
いよいよ、コンテナを起動します。以下のコマンドを実行してください。なお、コマンド内のオプションの意味は以下の通りです。
- `-i` : コンテナに対して標準入力から対話的にコマンドが実行できるようにする
- `-t` : コンテナ環境の中に擬似端末を割り当てる。
```bash
docker run -i -t ubuntu:latest
```
ターミナルが`root@<コンテナID>:/#`のようなプロンプトに変われば、コンテナ環境の起動に成功し、コンテナ内でコマンドを実行できる状態になっています。また、このとき、VS Codeの`Containers`の画面で、`ubuntu`というコンテナが追加されて、緑色の再生マークで表示されます。GUI上では以下の情報が確認できます。

- VS CodeのGUI上で表示されるコンテナにマウスカーソルを合わせると、コンテナの名前(指定のない場合はランダムで生成)とIDが表示されます。
- 起動中のコンテナはコンテナ左側の`>`のマークをクリックすると、コンテナ環境内のファイルをGUI上で確認することもできます。

#### コンテナ環境をコマンドで操作する
それでは、ここまでの手順で起動したコンテナ環境を、コマンド操作で管理してみましょう。

1. コンテナ環境のホスト名を確認する
    ```bash
    hostname
    ```
2. コンテナ環境のユーザー名を確認する 
    ```bash
    whoami
    ```
3. コンテナ環境の現在日時を確認する
    ```bash
    date
    ```
4. コンテナ環境を閉じる
    ```bash
    exit
    ```
    ※Dockerでは起動時に直接接続したターミナルを閉じるとコンテナも停止します。

#### VS CodeのGUIからの操作
コンテナ環境を閉じた後、VS Codeの`Containers`の画面で、先ほど起動したコンテナは停止状態で表示されています。右クリックから以下のような操作をGUI上で行うことができます。

| 操作内容 | 説明 |
| --- | --- |
| Start | 停止中のコンテナを起動する |
| Stop | 実行中のコンテナを停止する |
| Restart | 実行中のコンテナを再起動する |
| Attach Shell | コンテナに接続して、仮想環境のターミナルを開く |
| Remove | コンテナを削除する(削除後は他の操作はできません) |

一通りの操作をGUI上で確認した後、GUI上で`Remove`を選択してコンテナを削除してください。削除後、VS Codeの`Containers`の画面で、コンテナが消えていることを確認してください。

### 2. 条件を付けてコンテナを起動してみる

#### docker runコマンドのオプション
先程実行した`docker run`コマンドには、`-i`と`-t`の他にも様々なオプションを付けることができます。ここでは、いくつかの代表的なオプションを紹介します。
| 短い表記 | 長い表記 | 説明 | 引数 |
| --- | --- | --- | --- |
| -i | --interactive | コンテナに対して標準入力から対話的にコマンドが実行できるようにする | なし | 
| -t | --tty | コンテナ環境の中に擬似端末を割り当てる | なし |
| -d | --detach | コンテナをバックグラウンドで起動する | なし |
| (なし) | --name | コンテナに任意の名前を付ける | <コンテナ名> |
| (なし) | --rm | コンテナを停止した際に自動で削除する | なし |
| -p | --publish  | ホストのポートとコンテナのポートを紐付ける | <ホストのポート>:<コンテナのポート> |
| -v | --volume | ホストのディレクトリとコンテナのディレクトリを紐付ける | <ホストのディレクトリ>:<コンテナのディレクトリ> |
| -e | --env | コンテナ内の環境変数を設定する | <変数名>=<値> |
| (なし) | --env-file | コンテナ内の環境変数を設定するファイルを指定する | <環境変数ファイル> |

ここでは、`-i`、`-t`、`-d`、`--name`、`--rm`、`-v`の6つのオプションを使って、コンテナを起動してみます。以下のコマンドを実行してください。なお、引数が不要な一文字で指定するオプションは、まとめて指定することもできます。
```bash
docker run -itd --name my-ubuntu --rm -v ./volume:/workspace ubuntu:latest
```
起動後にVS Codeの`Containers`の画面で、`my-ubuntu`という名前のコンテナが起動していることを確認してください。さらに、コンテナ左側の`>`のマークを展開して、コンテナのファイルシステムを確認し、`/workspace`というディレクトリがホストの`./volume`と紐付いていることを確認してください。なお、ボリュームマウント中のファイルの更新は、ホスト側とコンテナ側の両方でリアルタイムに反映されます。

今回は、`-d`オプションを付けてコンテナをバックグラウンドで起動したため、ターミナルはすぐにホスト側のプロンプトに戻ります。コンテナの中に入るには、VS Codeの`Containers`の画面で該当のコンテナを右クリックし、`Attach Shell`を選択するか、以下のコマンドを実行してください。
```bash
docker exec -it my-ubuntu bash
```
※`docker exec`コマンドは、対象のコンテナ(コンテナ名またはコンテナID)と実行したい命令を引数に取ります。今回は、`bash`を引数にしているため、コンテナ内でbashシェルが起動します。また、`-i`と`-t`は`docker run`コマンドのオプションと同様に、標準入力の有効化と擬似端末の割り当てを行います。

`-d`オプションを付けて起動したコンテナは、ターミナルを閉じても停止しないため、`exit`コマンドでコンテナ内のターミナルを閉じても、コンテナは引き続き起動したままになります。コンテナを停止するには、VS Codeの`Containers`の画面で該当のコンテナを右クリックし、`Stop`を選択してください。自動で削除したい場合は`--rm`オプションを付けて起動できます（`--rm`と`-d`の組み合わせは運用上の注意があります）。

### 3. プログラミング言語の実行環境を構築する

#### Docker Hubの活用
先程は、ubuntuのイメージを使ってコンテナを起動しましたが、他にもインターネット上のDocker Hubには様々なイメージが用意されており、その中にはプログラミング言語の実行環境も存在します。ここでは、Pythonの実行環境を構築してみましょう。

まずは、以下のDocker Hubのページにアクセスしてください。

- [Docker Hub](https://hub.docker.com)

次に、検索欄に`python`と入力して検索してください。検索結果の中から、公式のPythonイメージを選択してください。公式イメージは、Docker Hubのページ上で`Official Image`というバッジが付いているものです。公式イメージのページにアクセスしたら、pullコマンドの例やイメージのタグの説明を確認して、実行するコマンドを決定して下さい。今回は、以下のコマンドを実行して、Python 3.14のイメージでコンテナを起動してみましょう。
※事前にpullをしていないためにローカルにイメージがない場合は、コマンド実行時に自動でDocker Hubからイメージがダウンロードされます。
```bash
docker run -itd -v ./volume:/workspace --name python-env python:3.14.6-trixie
```

### コンテナ内でPythonを実行する
先程起動したコンテナに接続して、Pythonを実行してみましょう。VS Codeの`Containers`の画面で、`python-env`というコンテナを右クリックし、`Attach Shell`を選択して、コンテナ内のターミナルを開いてください。そして、以下のコマンドを実行して、Pythonが正しく認識されていることを確認してください。
```bash
python --version
```
次にPythonのインタラクティブシェルを以下のコマンドから起動してください。
```bash
python
```
インタラクティブシェルが起動したら、以下のコードを入力して、Pythonの実行ができることを確認してみましょう。
```python
print("Hello, Docker!")
```

### マウントしたボリュームの活用
ボリュームマウントを行うことで、コンテナ環境にホスト側のファイルを反映させることができます。先程の手順で起動したコンテナでは、ホスト側の`./volume`というディレクトリが、コンテナ側の`/workspace`というディレクトリにマウントされていました。ここでは、マウントしたボリュームを活用して、Pythonのスクリプトを実行してみましょう。
まずは、ホスト側の`./volume`ディレクトリに、`hello.py`という名前のファイルを作成してください。そして、そのファイルの中に以下のコードを書いて保存してください。
```python
print("Hello from the mounted volume!")
```
次に、VS Codeの`Containers`の画面で、`python-env`というコンテナを右クリックし、`Attach Shell`を選択して、コンテナ内のターミナルを開いてください。そして、以下のコマンドを実行して、マウントしたボリューム内のPythonスクリプトが実行できることを確認してください。
```bash
python /workspace/hello.py
```
このような形でファイルをホスト側に置きながら、実行はコンテナ側で行うことで、ホストOSの環境を汚すことなく、コンテナ内で様々な実験や開発を行うことができます。ただし、実際の開発作業においては、より開発体験を向上させるために、VS CodeのDev Containers 拡張機能を活用して、ターミナルだけではなく、その他のVS Codeの機能もコンテナ内で利用できるようにして、あたかもローカル環境のように開発できる状態にすることが可能です。

### 4. データベース環境を構築する

#### PostgreSQLのコンテナイメージを探す
先程のPythonの実行環境の構築と同様に、データベースの環境もDocker Hubから取得することができます。ここでは、PostgreSQLというデータベースを使ってみましょう。まずは、以下のDocker Hubのページにアクセスして、公式イメージを見つけましょう。

- [Docker Hub](https://hub.docker.com)

公式イメージのページにアクセスしたら、pullコマンドの例やイメージのタグの説明があります。PostgreSQLのコンテナでは、以下のような`run`コマンドの例が紹介されています。このコマンドでは、`-e`オプションを使って、PostgreSQLの管理用ユーザーのパスワードを環境変数として設定しています。
```bash
docker run --name some-postgres -e POSTGRES_PASSWORD=mysecretpassword -d postgres
```

### PostgreSQLのコンテナを起動する
今回は上記で確認した記述に加えて、管理用ユーザーのユーザー名やタイムゾーンの設定を`-e`オプションで指定し、コンテナのポートをホストのポートに紐付けるために`-p`オプションを追加します。以下のコマンドを実行してください。
```bash
docker run --name my-postgres -e TZ=Asia/Tokyo -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=mysecretpassword -p 5432:5432 -d postgres
```
実行後にVS Codeの`Containers`の画面で、`my-postgres`という名前のコンテナが起動していることを確認してください。

### DB接続ツールからPostgreSQLに接続する
PostgreSQLのコンテナが起動したら、VS Code画面左側のアクティビティバーから`PostgreSQL`を選択して、PostgreSQLの拡張機能を開いてください。そして、CONNECTIONSの「接続の追加」マークをクリックして、新しい接続を追加してください。接続の設定では、以下の情報を入力してください。なお、ここでサーバー名を`localhost`と指定している理由は、`run`コマンド実行時にコンテナのポートをホストのポートに紐付けているためです。

| 項目 | 値 |
| --- | --- |
| サーバー名 | localhost |
| 認証の種類 | パスワード |
| ユーザー名 | postgres |
| パスワード | mysecretpassword |
| データベース名 | (空欄) |
| 接続名 | my-postgres(任意の名前で可) |

テスト接続のボタンを押して、チェックマークが表示されれば、接続成功です。接続が成功したら、「保存して接続」のボタンを押してください。CONNECTIONSの下に、先程保存した接続名で接続が追加されていることを確認してください。追加された接続をクリックして展開すると、データベースやテーブルの情報が表示されます。
Databaseから`postgres`を右クリックして、`New Query`を選択してください。そして、以下のSQLを入力して、実行してみましょう。
```sql
SELECT 
    'Hello, PostgreSQL!' AS テストメッセージ
    , CURRENT_TIMESTAMP AS 現在時刻
;
```
実行結果に`Hello, PostgreSQL!`と現在時刻が表示されれば、PostgreSQLのコンテナ環境が正しく構築されていることが確認できます。
なお、データベース環境の停止や削除は、ここまでで紹介したコンテナの停止や削除の方法と同様に、VS Codeの`Containers`の画面で該当のコンテナを右クリックして、`Stop`や`Remove`を選択することで行うことができます。

### コンテナ起動を設定ファイルで管理する
ここまでの手順では、`docker run`コマンドを直接実行してコンテナを起動してきましたが、実際の開発現場では、設定ファイルを使ってコンテナの起動を管理することが一般的です。設定ファイルには、イメージをカスタマイズするためのDockerfileやコンテナの起動を管理するdocker-compose.ymlなどがあります。

### docker-compose.yml を作成する
ここでは、docker-compose.ymlを使って、先程のPostgreSQLのデータベース環境を起動してみましょう。
以下の内容で、プロジェクトディレクトリ直下に`docker-compose.yml`という名前のファイルを作成してください。なお、`docker-compose.yml`では、起動したいコンテナをサービスとして定義する形式で記述します。今回は、PostgreSQLのコンテナを`my-postgres`というサービス名で定義しています。
```yml title="docker-compose.yml"
services:
  my-postgres:
    image: postgres:latest
    container_name: my-postgres
    environment:
      - TZ=Asia/Tokyo
      - POSTGRES_USER=${POSTGRES_USER}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    ports:
      - "5432:5432"
```
この時、`POSTGRES_USER`や`POSTGRES_PASSWORD`などの認証情報の値は、直接docker-compose.ymlに書くのではなく、同じ階層に配置する`.env`という名前のファイルに定義しておき、`gitignore`に追加しておくことで、これらの情報をGitHubなどのリモートリポジトリにアップロードしないようにするのが一般的です。このリポジトリには`.env.example`も用意しているので、必要に応じてコピーして`.env`を作成してください。`.env`ファイルの内容は以下の通りです。docker-compose.yml側の`${}`で囲まれた部分が、`.env`ファイルで定義した値に置き換えられます。
```env title=".env"
POSTGRES_USER=postgres
POSTGRES_PASSWORD=mysecretpassword
```
### docker-compose.yml からコンテナを起動する
設定ファイルを使ってコンテナを起動するには、`docker-compose.yml`が存在するディレクトリで、以下のコマンドを実行します。なお、`-d`オプションを付けることで、コンテナをバックグラウンドで起動することができます。なお、GUIからも操作可能で、エクスプローラーで`docker-compose.yml`を右クリックして、`Compose Up`を選択することでも、同様にコンテナを起動することができます。
```bash
docker compose up -d
```
実行後にVS Codeの`Containers`の画面で、`my-postgres`という名前のコンテナが起動していることを確認してください。先程と同様に、PostgreSQLの拡張機能から接続して、SQLクエリを実行してみましょう。

### docker-compose.yml からコンテナを終了する
`docker-compose.yml`で起動したコンテナを終了するには、以下のコマンドを実行します。なお、GUIからも操作可能で、エクスプローラーで`docker-compose.yml`を右クリックして、`Compose Down`を選択することでも、同様にコンテナを停止することができます。
```bash
docker compose down
```
実行後にVS Codeの`Containers`の画面で、`my-postgres`という名前のコンテナが停止していることを確認してください。

### docker-compose.ymlでできること
今回はPostgreSQLのコンテナを起動するためのdocker-compose.ymlを作成しましたが、docker-compose.ymlを使うと、複数のコンテナをまとめて管理したり、コンテナ同士のネットワーク設定を行ったりすることもできます。例えば、Webアプリケーションとデータベースの両方をコンテナで構築している場合に、両者を同じdocker-compose.ymlで管理することで、環境構築の手順を簡略化することができます。

## 5. まとめ
ここまでの内容で、Dockerを使ってコンテナ環境を構築するための基本的な操作方法や、Dockerが提供するコンテナ化の仕組みについて理解できたかと思います。Dockerを活用することで、開発環境や本番環境の構築を効率化し、環境差分によるトラブルを減らすことができます。さらに、Dockerfileやdocker-compose.ymlなどの設定ファイルを使って、環境構築の手順をコード化することで、再現性の高い環境を簡単に構築できるようになります。今日をきっかけにDockerを活用してください。

#### 1.コマンド操作
- コンテナの一覧表示
    ```bash
    docker ps
    ```
- コンテナの一覧表示(停止中も含む)
    ```bash
    docker ps -a
    ```

    ```bash
    docker stop <コンテナ名またはコンテナID>
    ```

    ```bash

    ```

    ```bash
    docker images
    ```
- イメージの削除
    ```bash

    ```
#### 2. Dockerfile
ハンズオンではコンテナの起動方法についての設定ファイルとして、docker-compose.ymlを紹介しましたが、Dockerfileを使ってイメージをカスタマイズすることも可能です。Dockerfileは、コンテナのイメージを作成するための手順を記述したファイルで、ベースとなるイメージの指定や、必要なパッケージのインストール、環境変数の設定などを行うことができます。Dockerfileを使うことで、より柔軟にコンテナ環境を構築することができます。以下はJava 21とGitが使用できるubuntuベースのカスタムイメージを作成するためのDockerfileの例です。
```Dockerfile title="Dockerfile"
# ベースイメージの指定
FROM ubuntu:latest

# タイムゾーンの設定
ENV TZ=Asia/Tokyo

# 必要なパッケージのインストール
RUN apt-get update && \
    apt-get install -y openjdk-21-jdk git && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

# コンテナが停止しないように終了しないコマンドを実行させる
CMD ["sleep", "infinity"]
```

#### 3. DevContainers
VS CodeのDev Containers拡張機能を活用することで、ターミナルだけではなく、VS Codeのその他の機能もコンテナ内で利用できるようにすることができます。Dev Containersを使うと、VS Codeの拡張機能やデバッガーなどもコンテナ内で動かすことができるため、より開発体験を向上させることができます。Dev Containersを利用するには、プロジェクトディレクトリに`.devcontainer`という名前のディレクトリを作成し、その中に`devcontainer.json`という設定ファイルを配置します。`devcontainer.json`の中で、使用するDockerイメージや参照するdocker-compose.ymlのパス、コンテナ環境内で使用したいVS Code設定や拡張機能を指定します。
